# Woman Up! / 娘们要战斗

## 作品说明（100-300 字）

《Woman Up!》是一款面向独居、异地、时差与低能量状态女性的异步支持型 Web 应用。作品关注的不是“替代真人关系”，而是当睡眠不足、轻症不适、焦虑、孤独和安全感下降等小问题开始堆积时，提供一种低负担、隐私友好、可立即执行的介入机制。用户通过 24 小时时间分配看见自己的照料结构，观看 1/2/5/10 年未来人生预测轨迹动画，获得个人状态反馈，并进入异步战队空间。系统队友承担轻量陪跑、提醒与通关反馈，帮助用户完成稳定呼吸、生存判断、睡眠前自我照料等微任务，同时鼓励连接同频的女性一起进步和互助。

## 面向的问题

独居、异地或跨时区生活的人，经常不是在“大危机”发生时才需要帮助，而是在很多小事连续堆积时逐渐失去能量，例如独居女性平时没有做好身体健康管理，没有培养强壮有力的身体、良好的饮食和作息，也没有养成运动习惯、学习格斗术、接触对抗性运动。

这导致在异地遇到危险，或者处于孤立无援的状态下时，人会陷入身心的崩溃，甚至面临人身安全威胁。

本作品试图解决的是：如何在不制造陪聊依赖、不索取过多隐私、不要求真人即时在线的前提下，让用户获得一种可执行、可记录、低压力的支持。

## 核心方案

- **24 小时时间分配测试**：帮助用户看见训练、学习、睡眠、饮食休息和消耗项之间的结构关系。
- **未来轨迹动画**：在结果揭晓前，用 1 年、2 年、5 年、10 年的叙事反馈强化长期后果感。
- **异步战队空间**：默认匹配系统队友，强调“不必同时在线”的轻量支持机制。
- **稳定呼吸关卡**：通过一次吸气与呼气完成低能量自我照料任务，适合睡前、轻症不适、焦虑或独处时使用。
- **生存智慧小游戏**：用低成本选择题训练风险识别和求助策略。
- **格斗术与生存技能图鉴**：把身体能力、安全知识、逃离策略和女性互助意识放在同一套成长路径里。

## 设计原则

- **异步支持**：双人游戏不要求双方同时在线。
- **低负担介入**：任务短、小、可完成，避免在低能量时增加压力。
- **隐私优先**：不需要获取用户隐私信息，保证女性用户在一个安全的环境下训练提升，并连接到同频的女性
- **不替代真人关系**：系统队友只承担提醒、陪跑、记录和反馈，不扮演亲密关系替代品。
- **提前介入**：在睡眠、焦虑、孤独和安全感问题变大之前，提供一个小动作入口。

## 使用的 API、模型、框架或数据

### API

- **Browser MediaDevices API**：使用 `navigator.mediaDevices.getUserMedia()` 进行可选摄像头预览。
- **Web Storage API**：使用 `localStorage` 存储 24 小时分配、训练状态、邀请码和小游戏进度。
- **html2canvas**：用于将结果卡片导出为截图。

### 模型

- 当前 Demo 不使用外部 AI 模型。
- 不依赖云端推理、聊天模型或远程推荐系统。

### 框架与库

- **React 18**
- **Vite 5**
- **TailwindCSS 3**
- **React Router**
- **Lucide React**
- **html2canvas**

### 数据

本项目使用本地 Demo 数据：

- S/A/B/C/D 档位规则
- 1/2/5/10 年轨迹文案
- 格斗术科普卡片
- 生存技能图鉴
- 生存智慧选择题
- 单人榜与双人榜 mock 数据

所有数据均在前端本地运行，无远程数据库。
